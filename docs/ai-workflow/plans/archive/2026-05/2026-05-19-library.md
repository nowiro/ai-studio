# Plan: library — swap mock AuthService for Keycloak provider

## Goal

Demonstrate the `AUTH_CONTEXT` swap promise end-to-end: replace `provideMockKeycloak()` with `provideKeycloak()` in `library`'s `main.ts` and prove role guards still work, with a Docker Compose Keycloak realm seeded with reader + librarian users.

## Context

After audit `library` (port 4206) uses `provideMockKeycloak({ initialRole })`. ADR-0013 designed the swap. `libs/keycloak-auth` exists with both providers.

## Scope

| In                                                               | Out                                  |
| ---------------------------------------------------------------- | ------------------------------------ |
| Compose file with Keycloak server + realm import                 | Production-grade Keycloak ops        |
| Library `main.ts` switched to `provideKeycloak()` behind env var | Removing mock provider (kept for CI) |
| E2E using real Keycloak (KEYCLOAK_ENABLED=true)                  | Hosted IdP                           |

## Risks & mitigations

- **Risk:** Keycloak not running locally → e2e blocks — Mitigation: env gate skips the spec when `KEYCLOAK_ENABLED` unset.
- **Risk:** Token-refresh edge cases — Mitigation: keycloak-js handles silent refresh; covered by integration test stub.

## Rollback

`KEYCLOAK_ENABLED` env defaults to false. Revert = unset env + remove compose file.
