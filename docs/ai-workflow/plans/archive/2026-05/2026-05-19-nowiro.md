# Plan: nowiro — landing redesign + analytics + docs/projects entry

## Goal

Refresh the `nowiro` landing app with the latest brand tokens, wire GA4 event tracking for the four hero CTAs, and create the missing `docs/projects/nowiro/` quartet so the app is documented like every other demo.

## Context

Per audit `apps/nowiro` runs as the brand entrypoint (`pnpm start:nowiro`). No `docs/projects/nowiro/` entry exists — it is the only first-party app without per-project docs.

## Scope

| In                                                            | Out                                         |
| ------------------------------------------------------------- | ------------------------------------------- |
| Visual refresh (Material 3 tokens + Tailwind v4 utilities)    | Marketing copy work                         |
| GA4 client (typed events, consent gate)                       | Server analytics                            |
| `docs/projects/nowiro/{README,business,technical,testing}.md` | Multi-page site (single-page landing stays) |

## Validation gate

```bash
pnpm affected:lint && pnpm typecheck && pnpm affected:test && pnpm affected:e2e && pnpm affected:build
```

## Risks & mitigations

- **Risk:** GA4 fires before consent — Mitigation: gate at service level (return early when `consent !== 'granted'`); E2E covers both states.
- **Risk:** docs/projects/nowiro adds parity expectations to other minimal apps (union-vault) — Mitigation: stand alone; address union-vault in its own plan.

## Rollback

GA4 client is opt-in via `provideGa4({ measurementId })`. Without it, no events. Doc files are additive.
