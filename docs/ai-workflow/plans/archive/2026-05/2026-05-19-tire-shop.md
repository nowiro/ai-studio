# Plan: tire-shop — migrate legacy checkout to shop-core/shop-ui

## Goal

Eliminate the duplicated cart + checkout code in `tire-shop` (predates the `shop-core`/`shop-ui` split) by migrating to the shared primitives used by bookstore / tools-shop / toy-shop, while keeping the existing tire-specific faceted filters.

## Context

Per 2026-05-19 audit `tire-shop` (`docs/projects/tire-shop`) ships independently at :4205. Its checkout is a copy of what shop-core/shop-ui later generalised. Documented as tech debt in `docs/projects/README.md`.

## Scope

| In                                                           | Out                                             |
| ------------------------------------------------------------ | ----------------------------------------------- |
| Swap local cart service → `ShopCartService` (libs/shop-core) | Tire-specific facets (size, season, brand) stay |
| Swap local checkout → `<ais-shop-checkout>` (libs/shop-ui)   | Visual identity (colours, hero) stays           |
| Migrate to `PRODUCT_LOOKUP` + `CART_STORAGE_KEY` DI tokens   | Backend integration                             |
| Remove now-orphan files post-migration                       | Re-orging the tire-shop feature libs            |

## Risks & mitigations

- **Risk:** Behavioural regression in checkout — Mitigation: existing E2E spec stays; new migration spec is additive; both must pass.
- **Risk:** Tire-specific cart-line metadata (e.g. tyre size) needs shop-core schema change — Mitigation: `CartLine` already has `meta: Record<string, unknown>`; map tire size into `meta.size`.

## Rollback

Single feature branch with deletions. Revert via `git checkout` restores prior copy of the local checkout/cart.
