# Plan: toy-shop — EU toy safety badges + age-gate consent

## Goal

Add CE / EN 71 / RoHS badge metadata to product cards and a parental-consent age gate (modal) before adding age-restricted toys (12+, 14+) to the cart. Both serve the demo's "real-world e-commerce concerns" promise.

## Context

Per audit `toy-shop` runs at :4210 on `shop-core` / `shop-ui` with age-gating + safety as documented theme. No real badges or consent UI yet.

## Scope

| In                                                                         | Out                                    |
| -------------------------------------------------------------------------- | -------------------------------------- |
| `safetyBadges: SafetyBadge[]` field on toy products                        | Real certificate registry              |
| Age-gate modal (Material 3 dialog) before add-to-cart for restricted items | Hard-blocked checkout (deterrent only) |
| Persistent age confirmation per session (localStorage TTL 24h)             | Identity verification                  |
| Audit log: each gate decision (granted/denied)                             | Government reporting                   |

## Risks & mitigations

- **Risk:** Modal becomes annoying — Mitigation: 24h session memory; tested in T007.
- **Risk:** ShopCartService change leaks into bookstore / tools-shop — Mitigation: feature-flag `provideAgeGate({ minAge })`; only toy-shop enables.

## Rollback

`provideAgeGate()` is opt-in via `main.ts`. Without it `addItem` works unchanged across all shops.
